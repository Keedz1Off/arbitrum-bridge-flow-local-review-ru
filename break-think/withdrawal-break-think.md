# Withdrawal Break Think

## burn(...)

```text
ИНВАРИАНТ
Токены должны быть сожжены на L2 перед выдачей токенов на L1.

ПОСЛЕДСТВИЯ
Это может привести к выдаче L1 токенов без burn на L2.
Bridge может выдать токены, которые не были обеспечены сожженными L2 токенами.
```

## finalizeWithdrawal(...)

```text
ИНВАРИАНТ
Только подлинное bridge-сообщение L2 -> L1 может выдать L1 токены.

ПОСЛЕДСТВИЯ
Attacker может создать spoofed message.
Это может привести к fake withdrawal finalization.
```

## release(...)

```text
ИНВАРИАНТ
Количество выданных токенов должно равняться количеству сожженных токенов.

ПОСЛЕДСТВИЯ
Bridge может выдать больше токенов, чем было реально сожжено на L2.
Это может привести к потере токенов из escrow и сломанному accounting.
```
